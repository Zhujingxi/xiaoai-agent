# XiaoAI Agent 轻量 Web 配置页设计

日期：2026-07-15

## 背景

`xiaoai-agent` 当前只在启动时从 `agent.yaml` 读取配置。修改 API Key、选择 ASR/语音后端或启用可选功能，都需要通过 SSH 编辑 YAML 并手动重启进程。

本设计在现有 Rust 程序内加入一个面向局域网的轻量配置页，让用户通过手机或电脑完成常用配置、查看基本调试信息，并按需重启 Agent 使配置生效。

## 已确认的约束

- Web 页面直接由音箱上的 `xiaoai-agent` 提供，不部署独立服务。
- 运行环境是资源有限的 ARMv7 Linux 音箱。
- 页面只在可信局域网内使用，不增加登录或鉴权。
- 页面聚焦常用设置；底层音频和设备命令主要用于只读调试。
- 保存配置后不热加载，也不改变当前运行中的服务。
- 用户通过独立按钮只重启 `xiaoai-agent`，不重启整台音箱。
- UI 以简洁、明确、易读、易用为目标，不使用复杂视觉效果或前端框架。
- 尽量避免音箱 NAND 的额外写入；只有用户主动保存时才写配置和备份。

## 目标

1. 在 `http://<speaker-ip>:8080` 提供单页配置界面。
2. 管理常用 API Key、服务地址、模型、后端选择和功能启用状态。
3. 对密钥脱敏，浏览器不能读取已保存密钥的原文。
4. 保存前执行与程序启动一致的配置校验。
5. 使用临时文件、备份和原子替换降低配置损坏风险。
6. 显示足够的只读运行信息和近期日志，帮助排查常见问题。
7. 通过按钮重启 Agent 进程，使磁盘上的新配置生效。
8. 保持较小的二进制、运行内存和前端资源开销。

## 非目标

- 不实现运行中热加载配置。
- 不实现用户、密码、会话、权限或 HTTPS。
- 不提供公网访问或远程管理方案。
- 不提供通用 YAML 编辑器。
- 不允许页面修改 shell 命令、原生音频库路径或大多数底层采样参数。
- 首版不提供外部服务连通性测试、日志下载、固件升级或整机重启。
- 不引入 React、Vue、Node.js 运行时、CSS 框架或前端构建服务。

## 技术方案

### 进程内 Web 服务

在现有 Tokio 运行时中启动 Axum Router，默认监听 `0.0.0.0:8080`。监听地址和端口通过 CLI 参数提供，默认值分别为 `0.0.0.0` 和 `8080`，不依赖 `agent.yaml`，避免配置保存接口修改自身监听入口。

前端由一个内嵌的 HTML 文件组成，CSS 和 JavaScript 直接包含在该文件中，并通过 `include_str!` 编译进二进制。运行时不需要静态资源目录，也不需要 Node.js。

Web 服务在主语音服务完成初始化后启动。通过页面生成的配置在写入前一定会经过完整校验；如果用户通过 SSH 手工写坏 YAML，程序可能无法启动 Web 页面，此时使用同目录的 `agent.yaml.bak` 通过 SSH 恢复。

### 模块边界

- `web/mod.rs`：创建 Router、Web 状态和服务生命周期。
- `web/api.rs`：定义配置、状态、日志和重启 HTTP 接口。
- `web/config_store.rs`：读取、脱敏、白名单更新、校验、备份和原子写入 YAML。
- `web/status.rs`：保存启动时间、KWS/会话状态、最近错误和固定长度内存日志。
- `web/index.html`：语义化单页表单和少量原生 CSS/JavaScript。

各模块通过明确的数据对象通信。API 层不能直接操作文件，页面也不能提交未列入白名单的任意 YAML 路径。

## 页面信息架构

页面使用系统字体、白色背景、清晰边框和少量状态色。桌面端设置合理的最大内容宽度，移动端按单列展示。不使用动画、渐变、装饰性大图、复杂阴影或图标库。

页面从上到下分为：

1. **运行状态**
   - Agent 是否就绪
   - 当前版本和运行时长
   - 当前语音模式、ASR 后端
   - 配置是否已保存但尚未重启
   - “保存配置”和“重启服务”主操作
2. **语音与模型**
   - `voice.runtime`
   - Qwen Realtime URL、API Key、模型和音色
   - `asr.provider`
   - OpenAI-compatible ASR 和 Realtime ASR 的地址、API Key、模型
   - Xiaomi AIVS 的 ASR-only 状态；危险的云执行开关不在常用页面开放
   - LLM 地址、API Key、模型、超时、最大输出和温度
3. **工具与可选功能**
   - 时区和天气配置
   - Tavily 搜索启用状态、地址和 API Key
   - Home Assistant MCP 启用状态、地址、Token 和超时
   - 音乐启用状态、Provider，以及 Navidrome/网易云对应字段
   - AirPlay 启用状态、名称、端口、密码和中断方式
4. **常规设置**
   - 连续对话闲置超时
   - 唤醒应答文本
   - LED 总开关
5. **高级与调试信息**
   - 默认折叠
   - 配置文件路径、KWS 状态、活动会话状态和已启用功能
   - 音频采样格式、VAD 阈值、底层设备命令和 LED 编号，只读显示
   - 最近错误和最近最多 200 行内存日志

选择器控制相关字段的可见性。例如只有 `native_qwen` 才显示 Qwen 字段，ASR 和音乐也只显示当前 Provider 的字段。隐藏字段不会被清空。

所有输入框都有可见标签和简短说明。错误信息显示在对应字段附近，状态不能只依赖颜色表达。页面维护“未保存”“已保存、需重启”“运行中”三种清晰状态。

## HTTP API

所有响应使用 JSON；HTML 只从 `/` 返回。请求体限制为 64 KiB，防止意外的大请求占用内存。

### `GET /api/config`

返回页面可编辑的白名单配置和只读高级配置。所有密钥只返回：

```json
{
  "configured": true,
  "masked": "••••abcd"
}
```

接口不返回密钥原文。`masked` 最多显示末尾四个字符；过短的值全部隐藏。

### `PUT /api/config`

提交白名单字段。普通字段携带目标值。密钥使用明确的更新语义：

```json
{
  "mode": "keep | replace | clear",
  "value": "仅 replace 时使用"
}
```

- `keep` 保留磁盘上的现有值。
- `replace` 使用新值。
- `clear` 明确清空。

页面默认发送 `keep`。仅在用户输入新密钥后发送 `replace`，点击单独的“清除”操作后发送 `clear`。

成功保存返回新的非敏感配置摘要和 `restart_required: true`。校验失败返回 `400` 和结构化字段错误；磁盘写入失败返回 `500`，原配置保持不变。

### `GET /api/status`

返回：

- 版本、启动时间、运行时长
- KWS 是否启动
- 当前是否存在活动语音会话
- 语音模式和 ASR 后端
- Web 搜索、Home Assistant、音乐、AirPlay 的启用状态
- 是否存在已保存但尚未重启的配置
- 最近一次运行错误摘要

页面每 5 秒刷新一次。重启等待期间改为每秒刷新，最长等待 30 秒。

### `GET /api/logs?limit=<n>`

返回内存环形缓冲区中的近期日志，`limit` 限制在 1 到 200。日志缓冲区固定容量，不从 NAND 反复读取日志文件。

### `POST /api/restart`

如果没有配置写入正在进行，返回 `202 Accepted`，然后在短暂延迟后执行进程替换。Linux 上使用当前可执行文件和原始启动参数替换当前进程映像，保留现有 PID、环境变量和日志重定向，只重启 Agent。

重启请求不执行音箱 reboot，也不恢复原生小爱音频路径。连续重启请求只接受第一个。

## 配置存储与校验

配置存储组件使用异步互斥锁串行化读取、保存和重启操作。

保存流程：

1. 读取当前 `agent.yaml` 为 `serde_yaml::Value`。
2. 只修改 API 白名单中的 YAML 路径。
3. 把候选 YAML 写入配置目录中的临时文件。
4. 使用 `AppConfig::load` 加载临时文件，执行与启动时相同的类型、路径和约束校验。
5. 校验失败时删除临时文件并返回字段错误。
6. 校验成功后把当前正式文件复制为 `agent.yaml.bak`。
7. 设置临时文件权限为 `0600`，同步文件内容后原子重命名为正式文件。
8. 标记 `restart_required`，但不修改内存中的运行配置。

白名单更新会保留页面未管理的字段和值。YAML 注释不保证保留，因此 `agent.example.yaml` 继续作为带注释的配置参考，运行文件由页面保存后以规范 YAML 表示。

## 运行状态与日志

增加一个小型共享状态对象：

- 使用 `Instant` 计算运行时长。
- 使用原子值记录 KWS 和活动会话状态。
- 保存最近一次错误的短文本。
- 使用受限的 `VecDeque` 保存最多 200 条 tracing 日志。

日志每条设置长度上限，避免第三方错误响应或异常文本无限占用内存。状态接口不返回 API Key、Token、密码、Cookie 或完整 Authorization header。写日志时继续遵循同样的脱敏要求。

## 错误处理

- 浏览器执行基本的必填项、数字范围和 URL 格式检查，但后端校验是最终依据。
- 字段错误以稳定字段路径返回，例如 `voice.qwen.api_key`。
- 保存失败不改变正式配置，也不设置 `restart_required`。
- 备份失败时停止保存，避免在没有恢复点的情况下替换配置。
- 重启按钮需要一次确认，并在保存进行中禁用。
- 重启后页面通过状态轮询判断恢复；30 秒内未恢复则提示用户通过 SSH 查看日志或恢复备份，不宣称重启成功。
- API 返回不包含 Rust 调试栈或密钥值；详细错误仅进入脱敏日志。

## 局域网安全边界

按用户要求不实现鉴权。服务监听全部网络接口，局域网内任何能访问 8080 端口的客户端都可以修改配置和重启 Agent。

即使没有鉴权，仍采用以下最小保护：

- 读取接口不返回密钥原文。
- 只允许白名单字段，不能通过 Web API 修改任意文件或 shell 命令。
- 严格限制请求体大小、日志数量和文本字段长度。
- 对所有写接口仅接受 JSON 和预期 HTTP 方法。
- 不提供执行命令、上传文件、读取任意路径或整机重启接口。

## 资源预算

- 一个 Axum Router 复用现有 Tokio 运行时。
- 一个内嵌 HTML 文件，不启动前端运行时或额外文件服务。
- 状态轮询使用普通短请求，不保持 WebSocket 连接。
- 日志缓冲固定 200 条，并限制单条长度。
- 不在后台周期性写 NAND。
- 保存和备份只在用户主动操作时发生。

## 测试与验证

### 单元测试

- 所有密钥读取均脱敏，响应中不存在原文。
- 密钥 `keep`、`replace`、`clear` 三种语义正确。
- 白名单更新不会改变隐藏的音频参数和设备命令。
- Provider 切换不会清空未显示的其他 Provider 配置。
- 无效 URL、端口、超时、Qwen 音频约束和必需密钥被拒绝。
- 临时文件校验失败时正式文件和备份不改变。
- 备份与原子替换成功，并保留 `0600` 权限。
- 日志环形缓冲遵守容量和单条长度限制。

### HTTP 集成测试

- `/` 返回内嵌页面。
- 配置、状态和日志接口返回正确 Content-Type 与状态码。
- 请求体过大、未知字段、错误方法和无效 JSON 被拒绝。
- 保存请求串行化，保存期间的重启请求被拒绝。
- 重启控制器通过依赖注入替换为测试实现，测试不会真的替换进程。

### 页面验证

- 桌面和手机宽度下保持清晰单列/受限宽度布局。
- 选择语音、ASR 和音乐 Provider 时字段显隐正确。
- 未修改的密钥保持 `keep`，清除操作必须明确确认。
- 保存状态、字段错误和重启等待状态不依赖颜色才能理解。
